# 실시간 토큰 잔량 표시

Claude Code(CLI)에서 **토큰 잔량을 상태줄(status line)에 실시간 표시**하는 방법. 두 가지를 함께 띄운다.

- **컨텍스트 창 잔량** — 대화가 담기는 200,000토큰(확장 모델 1,000,000토큰) 중 남은 양. 꽉 차면 자동 압축(compact)이 일어난다.
- **요금제 사용량 한도 잔량** — Pro/Max 구독의 5시간·7일 사용량 창 중 남은 양.

상태줄에 남은 양을 띄워 두면 언제 정리할지, 한도에 얼마나 근접했는지 눈으로 판단할 수 있다.

> 상태줄은 화면 하단에 상시 표시되는 줄로, 셸 스크립트를 실행해 그 출력을 보여준다. 스크립트는 매 어시스턴트 응답·`/compact`·권한 모드 변경 시점에 실행된다(300ms 디바운스). **상태줄 실행 자체는 API 토큰을 쓰지 않는다.**

---

## 방법 1: `/statusline` 명령 (가장 쉬움)

Claude Code에 자연어로 요청하면 스크립트 생성과 설정을 자동으로 처리한다.

```text
/statusline 모델 이름, 남은 컨텍스트 퍼센트, 5시간·7일 요금제 사용량 한도 잔량을 보여줘
```

---

## 방법 2: 직접 구성 (현재 적용된 설정)

아래가 지금 적용된 설정이다. **다른 PC에서도 이 두 파일만 그대로 만들면 동일하게 동작한다.** (`jq` 필요, 미설치 시 `sudo dnf install -y jq`.)

### 1. 스크립트

`~/.claude/statusline-command.sh`:

```bash
#!/bin/bash

input=$(cat)

model=$(echo "$input" | jq -r '.model.display_name')
remaining=$(echo "$input" | jq -r '.context_window.remaining_percentage // empty')
five=$(echo "$input" | jq -r '.rate_limits.five_hour.used_percentage // empty')
week=$(echo "$input" | jq -r '.rate_limits.seven_day.used_percentage // empty')

RESET=$'\033[0m'
CYAN=$'\033[36m'
YELLOW=$'\033[33m'
MAGENTA=$'\033[35m'

output="${CYAN}${model}${RESET}"

# 컨텍스트 잔량(%)
if [ -n "$remaining" ]; then
  ctx=$(printf '%.0f' "$remaining")
  output="${output} | ${YELLOW}Context ${ctx}%${RESET}"
fi

# 요금제 사용량 한도 잔량 = 100 − 소진율 (Pro/Max, 첫 API 응답 이후에만 존재)
limit_str=""
if [ -n "$five" ]; then
  fused=$(printf '%.0f' "$five")
  fpct=$((100 - fused))
  limit_str="5h ${fpct}% 남음"
fi
if [ -n "$week" ]; then
  wused=$(printf '%.0f' "$week")
  wpct=$((100 - wused))
  if [ -n "$limit_str" ]; then
    limit_str="${limit_str}, 7d ${wpct}% 남음"
  else
    limit_str="7d ${wpct}% 남음"
  fi
fi

if [ -n "$limit_str" ]; then
  output="${output} | ${MAGENTA}${limit_str}${RESET}"
fi

printf '%s\n' "$output"
```

출력 예: `Claude Opus 4.8 | Context 72% | 5h 65% 남음, 7d 88% 남음` (모델명 청록, 컨텍스트 노랑, 요금제 마젠타 색)

### 2. 설정 등록

`~/.claude/settings.json`의 `statusLine`에 이 스크립트를 지정한다(기존 다른 설정은 그대로 두고 이 키만 추가). `bash`로 실행하므로 스크립트에 실행 권한(chmod)은 없어도 된다.

```json
{
  "statusLine": {
    "type": "command",
    "command": "bash ~/.claude/statusline-command.sh"
  }
}
```

설정은 자동으로 다시 읽히지만, 화면에는 **다음 상호작용부터** 반영된다.

## 진행 바로 보기

퍼센트를 10칸짜리 막대로 그리는 예시다. `used_percentage`(사용률)를 채운다.

`~/.claude/statusline.sh`:

```bash
#!/bin/bash
input=$(cat)

MODEL=$(echo "$input" | jq -r '.model.display_name')
PCT=$(echo "$input" | jq -r '.context_window.used_percentage // 0' | cut -d. -f1)

BAR_WIDTH=10
FILLED=$(( PCT * BAR_WIDTH / 100 ))
EMPTY=$(( BAR_WIDTH - FILLED ))
BAR=""
[ "$FILLED" -gt 0 ] && printf -v FILL "%${FILLED}s" && BAR="${FILL// /▓}"
[ "$EMPTY" -gt 0 ]  && printf -v PAD  "%${EMPTY}s"  && BAR="${BAR}${PAD// /░}"

echo "[$MODEL] $BAR ${PCT}% 사용 / $(( 100 - PCT ))% 남음"
```

출력 예: `[Claude Opus 4.8] ▓▓▓░░░░░░░ 25% 사용 / 75% 남음`

## JSON 필드 (컨텍스트 관련)

stdin으로 들어오는 JSON 중 토큰·컨텍스트에 쓰는 필드다.

| 필드 | 의미 |
|:-----|:-----|
| `context_window.context_window_size` | 컨텍스트 창 최대 토큰. 기본 200000, 확장 모델 1000000. |
| `context_window.total_input_tokens` | 현재 컨텍스트에 담긴 입력 토큰 합(캐시 읽기·쓰기 포함). |
| `context_window.total_output_tokens` | 최근 응답의 출력 토큰. |
| `context_window.used_percentage` | 사용률(%) — 입력 토큰 기준으로 미리 계산됨. |
| `context_window.remaining_percentage` | **남은 비율(%)** — 미리 계산됨. |
| `context_window.current_usage` | 최근 API 호출의 토큰 세부(`input_tokens`, `output_tokens`, `cache_creation_input_tokens`, `cache_read_input_tokens`). |
| `exceeds_200k_tokens` | 최근 응답의 총 토큰이 200k를 넘었는지(고정 임계값). |

주의할 점:

- `total_input_tokens`/`total_output_tokens`가 **현재 컨텍스트 사용량**을 뜻하는 것은 **v2.1.132 이상**부터다. 그 이전 버전에서는 세션 누적값이었다.
- `context_window.current_usage`·`used_percentage`·`remaining_percentage`는 세션 첫 API 호출 전, 그리고 `/compact` 직후 다음 호출 전까지 **`null`** 일 수 있다. 위 스크립트처럼 `// empty`로 방어해 해당 항목을 생략하거나 `// 0`/`// 100` 폴백을 둔다.
- `used_percentage`는 출력 토큰을 빼고 **입력 토큰만**(`input_tokens + cache_creation_input_tokens + cache_read_input_tokens`)으로 계산된다. 직접 계산할 때도 같은 공식을 써야 값이 맞는다.

## 요금제 사용량 한도 잔량 (Pro/Max)

위 스크립트의 **② 부분**이 이걸 표시한다. `rate_limits`는 **Claude.ai 구독자(Pro/Max)** 세션에서 **첫 API 응답 이후에만** 나타나며, 5시간·7일 창이 각각 독립적으로 존재하거나 없을 수 있다(없으면 `// empty`로 건너뛴다). 필드 값은 **소진율**이므로 **잔량 = 100 − 소진율**로 계산한다.

| 필드 | 의미 |
|:-----|:-----|
| `rate_limits.five_hour.used_percentage` | 5시간 창 소진율(0–100). 잔량 = 100 − 값. |
| `rate_limits.seven_day.used_percentage` | 7일 창 소진율(0–100). 잔량 = 100 − 값. |
